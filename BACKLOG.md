# Product Backlog

Future product ideas that are not part of the active implementation plan yet.

## Future Features

- [ ] Mirror the heart on the main app screen.
  - Show the same emotional heart experience inside the launcher Activity, not only on the home-screen widget.
  - Keep the in-app heart useful for setup, testing, and users who have not placed the widget yet.

- [ ] Add lightweight pairing options.
  - QR code pairing.
  - Local wireless / Bluetooth-assisted pairing.
  - Invite link / deep link pairing.
  - Compare the options before implementation and pick the simplest path that does not weaken pairing safety.

- [ ] Add heart customization.
  - Let the user change the heart's look or mood.
  - Keep customization compatible with pre-rendered widget frames.
  - Decide later whether a custom heart is shown locally only or also to the paired partner.

- [ ] Add tap timeline / heartbeat graph.
  - Use the X axis as a time scale.
  - Show spikes like a heartbeat rhythm at the moments when the partner tapped.
  - Consider two rhythm lines, one for each person.
  - Explore overlaying both lines on one graph so both histories can be compared.

- [ ] Evaluate showing own unread taps on the widget.
  - Current widget badge is for received unread partner taps.
  - Consider whether the widget should also show local sent taps that the partner has not acknowledged.
  - Decide whether this needs a second badge, a different visual state, or should stay out of the widget.
